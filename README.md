# Amazon Sales SQL Analysis

## Project Overview

This SQL project analyzes Amazon sales transaction data to uncover business insights related to customer behavior, product performance, revenue generation, taxation, and branch operations.

The project focuses on solving business problems using SQL queries involving:

- Aggregations
- GROUP BY analysis
- CASE statements
- Subqueries
- Sorting & ranking
- Time-based analysis
- Customer segmentation
- Revenue analysis

---

## Business Problem

Retail businesses need data-driven insights to understand:

- Which product categories generate the highest revenue
- Customer purchasing behavior
- Payment preferences
- Branch-wise performance
- Sales trends over time
- Customer satisfaction and ratings

This project helps uncover operational and sales insights that can support better inventory planning, customer targeting, and revenue optimization.

## Key Business Questions Solved

### Revenue & Sales Analysis

- Which product line generates the highest sales?
- Which city generates the lowest sales?
- Which month generated the highest revenue?
- Which product line contributes the highest VAT?

### Customer Analysis

- Which customer type contributes the highest revenue?
- Which gender shops the most?
- What are the most preferred payment methods?

### Operational Analysis

- Which branches sell above average quantity?
- What are the peak sales timings?
- Which weekdays receive the highest ratings?

### Product Performance

- Which product categories are performing well?
- Which product lines have the highest ratings?

---

## Sample Insights

- Certain product lines contribute significantly higher revenue than others.
- Customer purchasing behavior varies across branches and cities.
- Payment methods show clear customer preference patterns.
- Higher-rated sales periods can indicate stronger customer experience.
- Some branches consistently perform above average in quantity sold.
